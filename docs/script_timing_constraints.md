# Script and Subtitle Timing Constraints

本文档约束短视频脚本、字幕和单张图片展示节奏，目标是避免“每张图讲太多、信息密度低、节奏拖”的问题。

## 1. 目标时长

针对网站功能种草、产品功能展示、AI 生成结果展示类视频，默认目标是：

```text
首选总时长：15-20s
可接受上限：24s
超过 24s：需要压缩脚本或拆成多条视频
```

如果一批素材原本可剪出 40s，通常应优先压缩到 15-20s，而不是把所有解释都塞进单条视频。

## 2. 单图展示时长

每张图片或每个视觉 beat 的推荐展示时长：

```text
普通网站截图：1.5-2.0s
参数页 / 复杂 UI：1.8-2.4s
结果图 / 行业案例图：1.4-2.0s
收口 CTA：1.2-1.8s
```

硬性上限：

```text
单图连续讲解尽量不超过 3.0s
超过 3.0s 通常说明文案太长，或者应拆成多个视觉 beat
```

## 3. 单段字幕字数

每段字幕应符合：

```text
优先：10-14 个中文口播字
软上限：16 个中文口播字
硬上限：22 个中文口播字
```

不要让一张图承载一整句长解释。每张图只表达：

```text
1 个结论 + 1 个关键词
```

例如：

```text
差：参数页里填品牌名和行业后，点击开始生成，就能让AI快速产出方案。
好：填品牌和行业，点生成。
```

## 4. 句式约束

避免这些结构：

```text
说明性连接词过多：适合、便于、能够、从而、并且、同时
一句话多重目的：路径清晰 + 上手快 + 不用培训
每段都是完整长句，缺少快打点
```

推荐风格：

```text
短句
动词开头
结果词收尾
少形容词，多动作词
```

## 5. 推荐脚本结构

功能种草视频建议结构：

```text
开场 1 句：平台定位 / 痛点
路径 1 句：从哪里进入
参数 1 句：怎么填
结果 4-5 句：每图一个行业标签 + 一个价值词
收口 1 句：一句 CTA
```

示例：

```text
开场：做品牌 LOGO，别从零画。
路径：打开文生图，选 LOGO。
参数：填品牌和行业，点生成。
结果1：交通出行，稳重清晰。
结果2：潮玩娱乐，更有张力。
结果3：母婴服务，温和亲切。
结果4：科技办公，简洁理性。
收口：多行业方案，一次跑完。
```

## 6. 校验脚本

新增校验脚本：

```bash
python scripts/check_subtitle_density.py \
  --subtitle cases/<case>/subtitle_track.json \
  --report cases/<case>/output/reports/subtitle_density_report.json \
  --json
```

它会检查：

```text
总时长是否超过 20s / 24s
单段字幕字数是否超过 16 / 22
单段展示时长是否超过 2.4s / 3.0s
口播速度是否过快
```

默认规则适合短视频功能种草场景。特殊长视频可以通过参数放宽，但不应作为默认行为。

## 7. 当前问题样例

如果一条视频是 8 段字幕、总时长约 41s、每段 30-36 字、每段 4.6-5.8s，那么它会被判定为：

```text
总时长过长
单图解释过重
每段字幕字数过多
视觉节奏拖慢
```

正确方向是保留素材，但压缩文案：

```text
40s 解释型视频 -> 15-20s 种草型视频
每段 30+ 字 -> 每段 10-16 字
每图 5s 左右 -> 每图 1.5-2.4s
```
