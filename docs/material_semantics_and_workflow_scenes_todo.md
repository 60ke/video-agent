# 素材语义与功能场景扩展 TODO

状态：`deferred`

本文档记录已经确认、但本阶段暂不进入实现的两项工作。当前视频链路继续使用现有资产角色和镜头模板；后续启动本 TODO 时无需保留旧模型兼容性。

## TODO 1：重建素材角色与关系语义

### 背景

当前 `assets/results` 中的图片通常按目录和文件名统一注册为 `result_image`。这种方式无法区分实景输入、风格参考、首次生成结果、局部编辑结果和实景融合结果，可能导致 Planner 把输入素材描述成最终效果。

### 目标

让 Planner 先理解素材在真实工作流中的角色和上下游关系，再生成讲述文案和镜头计划。

### 计划模型

建议至少支持以下角色：

- `real_scene_input`：现场照片或实景输入。
- `style_reference_input`：风格、构图或内容参考图。
- `generated_result`：首次生成结果。
- `edited_result`：局部编辑或重绘结果。
- `real_scene_fusion_result`：设计图与现场照片的融合结果。
- `plane_result`：平面图结果。
- `rendered_3d_result`：三维场景结果。
- `hd_result`：高清派生结果。

素材关系字段：

- `relationship_group_id`：同一次创作或同一条工作流的分组标识。
- `parent_asset_ids`：当前素材直接依赖的上游素材。
- `workflow_step`：素材在工作流中的阶段。
- `content_verified`：素材角色和内容是否经过视觉模型或人工确认。

### 约束

- 不再仅凭目录或“结果图”文件名判断素材角色。
- 输入素材不得承担“生成效果”类事实声明。
- 前后对比必须来自同一个 `relationship_group_id`。
- Planner 不得在缺失关系证据时自行拼接“参考图 -> 结果图”。
- 文件名保留中文语义，但结构化关系不能只存在于文件名中。

### 验收标准

- 实景参考图不会被选为生成结果。
- Planner 能明确区分“实景输入”“风格参考”和“最终结果”。
- 任意编辑、高清化、合成或三维派生结果均可追溯到父素材。
- 文案在素材选择之后生成，并准确描述已注册的素材关系。

## TODO 2：扩展真实功能工作流场景

### 背景

当前视频主要使用首页、功能入口、参数页和独立结果图，尚未充分利用网站已经存在的后续操作和前后状态，导致画面信息量不足、产品能力讲述不完整。

### 已确认能力

文化墙链路已确认包含：

- 实景图输入与参考图输入。
- 参数填写与首次生成。
- 局部编辑。
- 实景合成。
- 高清图生成。
- 平面图生成。
- 平面图转 3D 和 PNG 实景融合。

AI 图片工具已确认包含：

- 无痕改字、物体消除和空白填充。
- 背景替换、阴影编辑、尺寸调整和图片修复。
- 多图合一、智能配色和全能改色。
- 商品精修、打光、回正、试穿和姿势替换等垂直能力。

### 计划场景模板

- `reference_to_result`：参考图 -> 生成结果。
- `real_scene_to_fusion`：现场照片 -> 实景融合结果。
- `result_to_local_edit`：生成结果 -> 编辑区域/描述 -> 重绘结果。
- `result_to_hd`：普通结果 -> 高清结果。
- `result_to_plane_to_3d`：效果图 -> 平面图 -> 三维场景。
- `image_edit_before_after`：原图 -> 编辑动作 -> 编辑后图。
- `multi_reference_to_result`：多张参考图 -> 合并结果。
- `variation_gallery`：同一输入 -> 多个候选方案。

### 编排原则

- 先选工作流关系，再写文案，最后编译镜头和卡点。
- 每个场景必须绑定真实存在的输入、操作状态和输出。
- 画面变化应与短语锚点共用时间基准。
- 前后对比优先使用切换、擦拭或短暂交叉淡化，不使用无依据的局部放大。
- 代码中存在但浏览器不可见、不可稳定操作的能力不得进入正式视频。

### 验收标准

- 每种模板至少有一条完整、可追溯的素材关系样例。
- 字幕描述与当前画面角色一致。
- 同一工作流内输入、操作和结果不会跨功能或跨任务混用。
- 15 至 24 秒短视频中可使用多个状态增加信息量，同时保持语音、字幕和视觉动作卡点一致。

## 启动条件

满足以下条件后再进入实现：

1. 现有 V3 主链路的文字密集页面形变和基础音效配置稳定。
2. 网站素材采集能够提供输入、结果及必要的任务关联信息。
3. 至少选择一个功能作为完整工作流样板，并准备真实的前后素材。
